# 베로로(Veroro) 기능 명세서

**문서 버전:** v1.0  
**작성일:** 2026-06-21  
**서비스명:** 베로로(Veroro)  
**플랫폼:** iOS / Android (React Native + Capacitor)

---

## 서비스 개요

베로로는 보호자가 반려동물의 품종, 나이, 체중, 알러지, 건강 고민을 등록하면 사료/간식/영양제/생활용품을 맞춤 추천하고, 상품의 성분과 영양 정보를 분석해주는 반려동물 맞춤형 사료 성분 분석 앱이다.

---

## 우선순위 정의

| 등급 | 기준 |
|------|------|
| P0 | 서비스 핵심 가치 전달에 필수. 미구현 시 출시 불가 |
| P1 | 사용자 유지에 필요. MVP 이후 단계에서 필수 |
| P2 | 경험 향상. 안정화 후 구현 |
| P3 | 차별화 요소. 로드맵 후순위 |

## 1. 회원 기능

---

### 1-1. 이메일 회원가입

- **목적:** 사용자가 이메일과 비밀번호로 계정을 생성하여 개인화된 서비스를 이용할 수 있게 한다.
- **사용자 시나리오:**
  1. 사용자가 앱을 처음 실행한다.
  2. "회원가입" 버튼을 탭한다.
  3. 이메일, 비밀번호, 비밀번호 확인을 입력한다.
  4. 이용약관 및 개인정보처리방침에 동의한다.
  5. "가입하기" 버튼을 탭한다.
  6. 이메일 인증 메일이 발송된다.
  7. 인증 완료 후 로그인 상태로 진입한다.
- **입력값:**
  - 이메일 (필수, 형식 검증)
  - 비밀번호 (필수, 8자 이상, 영문+숫자+특수문자 1개 이상)
  - 비밀번호 확인 (필수, 위와 일치)
  - 이용약관 동의 (필수)
  - 개인정보처리방침 동의 (필수)
  - 마케팅 수신 동의 (선택)
- **출력값:**
  - 회원가입 성공 → 이메일 인증 안내 화면 전환
  - 인증 완료 → 마이펫 등록 온보딩으로 이동
  - 가입 실패 → 항목별 인라인 오류 메시지
- **예외 케이스:**
  - 이미 가입된 이메일: "이미 사용 중인 이메일입니다." 안내
  - 이메일 형식 불일치: 실시간 인라인 오류 표시
  - 비밀번호 정책 미충족: 항목별 오류 메시지
  - 비밀번호 불일치: "비밀번호가 일치하지 않습니다."
  - 인증 메일 미수신: 재발송 버튼 제공 (60초 쿨다운)
  - 인증 링크 만료 (24시간): 재발송 안내
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함

---

### 1-2. 이메일 로그인

- **목적:** 가입된 이메일 계정으로 인증하여 서비스에 접근한다.
- **사용자 시나리오:**
  1. 사용자가 로그인 화면에서 이메일과 비밀번호를 입력한다.
  2. "로그인" 버튼을 탭한다.
  3. 인증 성공 시 홈 화면으로 이동한다.
- **입력값:**
  - 이메일
  - 비밀번호
  - 자동 로그인 여부 (선택)
- **출력값:**
  - 로그인 성공 → 홈 화면 (또는 이전 화면)
  - 로그인 실패 → 오류 메시지
- **예외 케이스:**
  - 이메일 미가입: "가입되지 않은 이메일입니다."
  - 비밀번호 불일치: "비밀번호가 올바르지 않습니다."
  - 5회 연속 실패: 30분 잠금, 비밀번호 재설정 유도
  - 이메일 인증 미완료 계정: 인증 메일 재발송 안내
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함

---

### 1-3. 소셜 로그인

- **목적:** 카카오, 애플, 구글 계정을 이용해 회원가입 및 로그인 절차를 간소화한다.
- **사용자 시나리오:**
  1. 로그인 화면에서 소셜 로그인 버튼(카카오/애플/구글)을 탭한다.
  2. 해당 플랫폼 인증 화면으로 이동한다.
  3. 인증 완료 후 최초 가입이면 이용약관 동의 후 온보딩으로 이동한다.
  4. 기존 계정이면 홈 화면으로 바로 이동한다.
- **입력값:**
  - 소셜 플랫폼 선택 (카카오 / 구글 / 애플)
  - 각 플랫폼 OAuth 토큰
  - 이용약관 동의 (최초 가입 시)
- **출력값:**
  - 가입/로그인 성공 → 홈 또는 온보딩 화면
  - 소셜 인증 실패 → 오류 메시지 + 이메일 로그인 안내
- **예외 케이스:**
  - 동일 이메일로 다른 소셜 계정 가입 시도: 기존 연결 계정 안내 및 계정 연동 옵션 제공
  - 소셜 플랫폼 서비스 장애: "잠시 후 다시 시도해주세요."
  - 애플 로그인 시 이메일 비공개 선택: 임시 이메일로 계정 생성 처리
- **우선순위:** P1
- **MVP 포함 여부:** ✅ 포함 (카카오 우선, 구글/애플은 심사 조건 필수)

---

### 1-4. 로그아웃

- **목적:** 현재 기기에서 인증 세션을 종료한다.
- **사용자 시나리오:**
  1. 설정 > "로그아웃" 탭한다.
  2. 확인 다이얼로그가 표시된다.
  3. 확인 시 로그아웃 처리 후 로그인 화면으로 이동한다.
- **입력값:** 로그아웃 확인 액션
- **출력값:** 로컬 세션 토큰 삭제 → 로그인 화면으로 이동
- **예외 케이스:**
  - 네트워크 오류: 로컬 토큰만 삭제하고 로그아웃 처리
  - 자동 로그인 설정 해제 여부 선택 옵션 제공
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함

### 1-5. 회원 탈퇴

- **목적:** 사용자가 계정과 관련 데이터를 삭제하고 서비스를 탈퇴할 수 있게 한다.
- **사용자 시나리오:**
  1. 설정 > 계정 관리 > "회원 탈퇴"를 탭한다.
  2. 탈퇴 안내 (데이터 삭제 범위, 복구 불가 안내)를 확인한다.
  3. 탈퇴 사유를 선택한다 (선택).
  4. 비밀번호 또는 소셜 재인증을 완료한다.
  5. "탈퇴하기" 버튼을 탭한다.
  6. 탈퇴 처리 완료 후 온보딩 화면으로 이동한다.
- **입력값:**
  - 탈퇴 사유 (선택, 단일 선택)
  - 재인증 (비밀번호 or 소셜 토큰)
  - 탈퇴 확인 체크박스
- **출력값:**
  - 계정 및 개인 데이터 즉시 삭제 처리 (법적 보관 의무 데이터 제외)
  - 탈퇴 완료 화면 → 온보딩 화면
- **예외 케이스:**
  - 재인증 실패: 탈퇴 불가, 오류 안내
  - 탈퇴 처리 중 네트워크 오류: 재시도 안내, 데이터 불완전 삭제 방지
  - 법적 보관 의무 데이터(거래 기록 등)는 별도 고지 후 보관
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함 (앱스토어 심사 필수 요건)

### 1-6. 비밀번호 재설정

- **목적:** 비밀번호를 잊은 사용자가 이메일을 통해 재설정할 수 있게 한다.
- **사용자 시나리오:**
  1. 로그인 화면에서 "비밀번호를 잊으셨나요?" 탭한다.
  2. 가입된 이메일을 입력한다.
  3. 재설정 메일 발송 완료 안내를 받는다.
  4. 메일의 링크를 탭하면 새 비밀번호 입력 화면으로 이동한다.
  5. 새 비밀번호 입력 및 확인 후 변경 완료한다.
- **입력값:**
  - 이메일
  - 새 비밀번호 (정책 준수)
  - 새 비밀번호 확인
- **출력값:**
  - 재설정 메일 발송 성공 안내
  - 비밀번호 변경 완료 → 로그인 화면
- **예외 케이스:**
  - 미가입 이메일: 보안상 동일 문구("메일을 확인해주세요") 표시 (이메일 존재 여부 노출 방지)
  - 재설정 링크 만료 (1시간): 재발송 안내
  - 기존 비밀번호와 동일: "이전 비밀번호와 다른 비밀번호를 입력해주세요."
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함

### 1-7. 둘러보기 모드

- **목적:** 회원가입 없이 앱의 핵심 기능을 체험하게 하여 가입 전환율을 높인다.
- **사용자 시나리오:**
  1. 로그인 화면에서 "둘러보기" 버튼을 탭한다.
  2. 회원가입 없이 홈, 검색, 상품 상세, 성분 분석을 볼 수 있다.
  3. 마이펫 맞춤 기능은 비활성화 또는 샘플 데이터로 제공한다.
- **입력값:** 없음 (게스트 세션 자동 생성)
- **출력값:** 제한된 기능의 홈 화면 진입
- **예외 케이스:**
  - 앱 재실행 시 게스트 세션 유지 여부: 세션 유지 (디바이스 로컬 저장)
  - 둘러보기 상태에서 딥링크 접근 시 해당 화면으로 이동 후 필요 시 가입 유도
- **우선순위:** P1
- **MVP 포함 여부:** ✅ 포함

### 1-8. 둘러보기 중 회원가입 유도

- **목적:** 게스트 사용자가 찜/비교/마이펫 저장 등 회원 전용 기능에 접근 시 자연스럽게 가입을 유도한다.
- **사용자 시나리오:**
  1. 게스트 상태에서 상품 찜 버튼을 탭한다.
  2. "이 기능은 회원 전용입니다. 가입하시면 찜 목록을 저장할 수 있어요." 바텀시트가 나타난다.
  3. "가입하기" 또는 "나중에" 선택지를 제공한다.
  4. 가입하기 선택 시 회원가입 화면으로 이동하고 가입 완료 후 원래 화면으로 복귀한다.
- **입력값:** 회원 전용 기능 액션 트리거 (찜, 비교함 추가, 마이펫 저장, 분석 이력 저장)
- **출력값:** 회원가입 유도 바텀시트 또는 팝업
- **예외 케이스:**
  - "나중에" 선택 후 동일 세션에서 재시도: 동일 유도 화면 재표시
  - 이미 로그인 중인 상태에서 해당 기능 접근 시 유도 화면 미표시
- **우선순위:** P1
- **MVP 포함 여부:** ✅ 포함

## 2. 마이펫 기능

---

### 2-1. 펫 등록

- **목적:** 반려동물의 정보를 등록하여 맞춤 추천과 분석의 기반 데이터를 확보한다.
- **사용자 시나리오:**
  1. 마이펫 탭 또는 온보딩에서 "펫 추가" 버튼을 탭한다.
  2. 단계별 입력 폼(종 선택 → 기본정보 → 건강정보 → 식단정보)을 진행한다.
  3. 등록 완료 후 해당 펫을 기반으로 한 맞춤 홈 화면으로 이동한다.
- **입력값:**
  - 이름 (필수, 최대 10자)
  - 종 구분: 강아지 / 고양이 (필수)
  - 품종 (검색 + 선택, "모름/믹스" 선택 가능)
  - 생년월일 (달력 선택) 또는 나이 직접 입력 (필수)
  - 성별: 수컷 / 암컷 (필수)
  - 중성화 여부: 완료 / 미완료 / 모름 (필수)
  - 체중 kg (필수, 소수점 첫째 자리)
  - 활동량: 낮음 / 보통 / 높음 (필수)
  - 알러지 (다중 선택, 성분 태그 검색 가능)
  - 건강 고민 (다중 선택, 카테고리 제공: 피부, 관절, 소화, 비만, 심장 등)
  - 현재 급여 중인 사료 (상품 검색 연동, 선택)
  - 선호 식단: 건식 / 습식 / 생식 / 혼합 (선택)
  - 예산: 월 예산 범위 설정 슬라이더 (선택)
  - 프로필 사진 (선택, 카메라 or 갤러리)
- **출력값:**
  - 펫 프로필 생성 완료
  - 마이펫 목록에 추가
  - 추천 엔진에 펫 데이터 반영 즉시 트리거
- **예외 케이스:**
  - 필수 항목 미입력: 해당 항목 하이라이트 및 완료 버튼 비활성화
  - 체중 이상값 (0 이하, 종별 최대치 초과): "정확한 체중을 입력해주세요." 안내
  - 생년월일이 미래 날짜: 입력 불가 처리
  - 이미지 업로드 실패: 기본 프로필 이미지로 대체, 재시도 안내
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함

---

### 2-2. 펫 수정

- **목적:** 반려동물의 정보 변경(체중, 건강 상태 등)을 반영하여 추천 정확도를 유지한다.
- **사용자 시나리오:**
  1. 마이펫 탭 > 펫 카드 > "수정" 버튼을 탭한다.
  2. 기존 데이터가 자동으로 채워진 수정 폼이 열린다.
  3. 변경할 항목을 수정 후 저장한다.
- **입력값:** 펫 등록과 동일한 항목 (기존 값 프리필)
- **출력값:**
  - 변경 사항 저장 완료 안내
  - 추천 엔진 재계산 트리거
- **예외 케이스:**
  - 수정 중 앱 이탈: "변경사항이 저장되지 않습니다. 나가시겠어요?" 다이얼로그
  - 저장 실패 (네트워크 오류): 로컬 임시 저장 후 재시도
- **우선순위:** P0

## 3. 상품 검색 기능

### 3-1. 키워드 / 브랜드 / 성분 검색

- **목적:** 사용자가 원하는 상품을 빠르게 찾을 수 있게 한다.
- **사용자 시나리오:**
  1. 검색 탭을 탭하거나 상단 검색바를 탭한다.
  2. 키워드(상품명), 브랜드명, 또는 성분명을 입력한다.
  3. 실시간 자동완성 제안이 표시된다.
  4. 결과 목록이 표시된다.
- **입력값:**
  - 검색어 (최소 1자)
  - 검색 유형: 상품명 / 브랜드 / 성분 (탭 전환 또는 자동 감지)
- **출력값:**
  - 검색 결과 상품 목록 (상품 카드: 이미지, 상품명, 브랜드, 가격, 궁합 점수, 등급)
  - 검색어 하이라이트
  - 결과 없음 시 "결과가 없습니다. 다른 검색어를 입력해보세요." + 추천 검색어 제공
- **예외 케이스:**
  - 특수문자 포함 검색어: 정규화 처리
  - 성분명 오타: 유사 성분 제안 ("혹시 이 성분을 찾으시나요?")
  - 검색 결과 0건: 관련 카테고리 상품 제안
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함

### 3-2. 카테고리 검색 및 필터

- **목적:** 다양한 조건으로 상품을 좁혀 원하는 상품을 효율적으로 탐색한다.
- **사용자 시나리오:**
  1. 검색 결과 또는 홈에서 필터 버튼을 탭한다.
  2. 원하는 조건을 다중 선택한다.
  3. "적용" 버튼을 탭하면 필터가 반영된 결과가 표시된다.
- **입력값:**
  - 카테고리: 사료 / 간식 / 영양제 / 생활용품
  - 종 구분: 강아지 / 고양이 / 공용
  - 나이: 퍼피(~1세) / 어덜트(1~7세) / 시니어(7세 이상) / 전연령
  - 건강 고민: 다중 선택 (피부, 관절, 소화, 비만, 심장, 신장 등)
  - 알러지 제외 성분: 마이펫 알러지 자동 적용 또는 수동 선택
  - 가격 범위: 최소~최대 슬라이더 (원 단위)
  - 등급: A++ / A+ / A / B / C (다중 선택)
  - 평점: 3점 이상 / 4점 이상 / 4.5점 이상
- **출력값:**
  - 필터 적용 상품 목록
  - 적용된 필터 칩 목록 표시 (개별 해제 가능)
  - 전체 결과 수 표시
- **예외 케이스:**
  - 복합 필터 결과 0건: 조건 완화 제안 ("알러지 제외 조건을 해제하면 N개 상품이 있어요.")
  - 마이펫 미등록 상태에서 알러지 필터 사용: 마이펫 등록 유도
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함

### 3-3. 정렬

## 4. 상품 상세 기능

### 4-1. 상품 기본 정보 표시

- **목적:** 사용자가 구매 결정에 필요한 상품 정보를 한 화면에서 확인할 수 있게 한다.
- **사용자 시나리오:**
  1. 검색 결과 또는 추천 목록에서 상품 카드를 탭한다.
  2. 상품 상세 화면에서 모든 정보를 확인한다.
  3. 필요시 구매 링크로 이동하거나 분석 리포트를 확인한다.
- **입력값:** 상품 ID
- **출력값:**
  - 상품 이미지 (다중 이미지 스와이프)
  - 브랜드명 (탭 시 브랜드 상품 목록으로 이동)
  - 상품명
  - 가격 (정가 / 할인가 구분)
  - 중량 (g 또는 kg)
  - 평점 (별점 시각화)
  - 리뷰 수
  - 카테고리 (사료/간식/영양제/생활용품)
  - 연령대 (퍼피/어덜트/시니어/전연령)
  - 종 구분 (강아지/고양이/공용)
  - 주요 성분 (상위 5가지, 태그 형태)
  - 전성분 (펼쳐보기)
  - 보증성분 (조단백, 조지방, 조섬유, 수분 등)
  - 칼로리 (kcal/100g 또는 kcal/kg)
  - 급여 가이드 (체중별 권장 급여량 표)
  - 구매 링크 (외부 쇼핑몰)
  - 궁합 점수 (마이펫 기준)
  - 성분 등급 배지
- **예외 케이스:**
  - 이미지 없는 상품: 기본 이미지 표시
  - 성분 정보 없는 상품: "성분 정보가 등록되지 않았습니다." + 정보 제공 요청 버튼
  - 구매 링크 만료: 링크 오류 감지 시 "현재 구매 링크를 확인 중입니다." 표시
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함

- **입력값:** 상품 ID, 유저 ID
- **출력값:**
  - 찜: 하트 아이콘 활성화 상태 변경, 찜 목록에 추가
  - 비교: 화면 하단 비교함 바에 상품 섬네일 추가 (최대 4개)
- **예외 케이스:**
  - 비로그인 상태: 회원가입 유도 바텀시트
  - 비교함 4개 초과 시도: "비교함은 최대 4개까지 추가할 수 있어요." + 기존 항목 교체 옵션

## 5. 성분 분석 기능

---

### 5-1. 성분 분석 리포트

- **목적:** 상품의 전성분을 파싱하고 안전성, 영양 균형, 펫 적합성을 평가하여 보호자가 이해하기 쉬운 리포트를 제공한다.
- **사용자 시나리오:**
  1. 상품 상세 또는 스캐너에서 분석을 요청한다.
  2. 분석 리포트 화면이 표시된다.
  3. 성분별 상태, 주의 성분, 추천/비추천 이유를 확인한다.
- **입력값:**
  - 전성분 텍스트 (DB 또는 OCR)
  - 마이펫 프로필 (선택, 맞춤 분석 시 필요)
- **출력값:**
  - 주요 단백질원 목록 (순서, 비중)
  - 탄수화물원 목록
  - 기능성 성분 목록 (오메가3, 프리바이오틱스 등)
  - 주의 성분 목록 (착색료, 보존료, 산화방지제 등)
  - 알러지 성분 충돌 결과 (마이펫 기준)
  - 성분별 상태 배지:
    - 🟢 안전: 검증된 안전 성분
    - 🟡 주의: 과량 시 문제 가능
    - 🟠 확인필요: 연구 결과 분분
    - 🔴 위험: 반려동물에 유해 가능성
  - 성분 안전도 점수 (0~100)
  - 영양 균형 점수 (0~100)
  - 최종 등급 (A++ / A+ / A / B / C)
  - 쉬운 설명 (보호자 언어로 요약: "이 사료는 ~ 때문에 추천합니다/주의하세요.")
  - 면책 고지: "이 정보는 의료적 진단을 대체하지 않습니다."
- **예외 케이스:**
  - 성분 정보 없음: "성분 정보가 없어 분석이 불가합니다." + 직접 등록 요청
  - OCR 인식률 저하로 파싱 불완전: 인식된 성분만 분석 후 "일부 성분이 누락되었을 수 있습니다." 안내
  - 미등록 성분 포함: "확인되지 않은 성분이 포함되어 있습니다." 표시
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함

## 6. 궁합 점수 기능

---

### 6-1. 궁합 점수 산출

- **목적:** 상품이 특정 반려동물에게 얼마나 적합한지를 0~100점으로 정량화하여 선택을 돕는다.
- **사용자 시나리오:**
  1. 마이펫이 등록된 사용자가 상품을 조회한다.
  2. 상품 카드 및 상세 화면에 궁합 점수가 표시된다.
  3. "점수 근거 보기"를 탭하면 항목별 반영 사유를 확인한다.
- **입력값:**
  - 마이펫 프로필 (종, 나이, 체중, 활동량, 알러지, 건강 고민)
  - 상품 데이터 (성분, 보증성분, 칼로리, 카테고리, 연령대, 평점, 리뷰 수, 가격, 용량)
- **출력값:**
  - 궁합 점수 (0~100, 소수점 반올림 정수)
  - 점수 등급 표시: 90+ "최고", 75~89 "좋음", 60~74 "보통", 60 미만 "주의"
  - 항목별 기여 점수 및 사유 표시

  **가중치 구조:**

  | 항목 | 가중치 | 설명 |
  |------|--------|------|
  | 알러지 충돌 여부 | 25% | 충돌 성분 1개라도 존재 시 대폭 감점 |
  | 건강 고민 적합성 | 20% | 건강 고민 대응 성분 포함 여부 |
  | 영양 균형 | 20% | 단백질/지방/섬유 균형 |
  | 연령/종 적합성 | 15% | 라벨상 대상 연령/종 일치 여부 |
  | 주의 성분 존재 여부 | 10% | 주의/위험 성분 수에 따라 감점 |
  | 리뷰/평점 | 5% | 평점 및 리뷰 신뢰도 반영 |
  | 가성비 | 5% | g당 가격 대비 성분 품질 |

- **예외 케이스:**
  - 마이펫 미등록: 궁합 점수 미표시, "마이펫을 등록하면 궁합 점수를 볼 수 있어요." 표시
  - 상품 성분 정보 없음: 궁합 점수 산출 불가, "성분 정보 부족으로 점수 제공 불가" 표시
  - 알러지 충돌 상품: 점수와 관계없이 빨간 경고 배지 우선 표시
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함

## 7. 급여량 계산 기능

---

### 7-1. 급여량 계산기

- **목적:** 반려동물의 신체 조건과 상품 칼로리를 기반으로 하루 권장 급여량을 계산해 보호자의 급여 실수를 줄인다.
- **사용자 시나리오:**
  1. 상품 상세 > "급여량 계산하기" 버튼을 탭한다.
  2. 마이펫 정보가 자동 불러와지며 수동 조정도 가능하다.
  3. 체중관리 목표를 선택한다.
  4. 계산 결과가 표시된다.
- **입력값:**
  - 펫 체중 (kg)
  - 나이 (개월 또는 년)
  - 중성화 여부
  - 활동량 (낮음/보통/높음)
  - 상품 칼로리 (kcal/100g)
  - 체중관리 목표: 감량 / 유지 / 증량
- **출력값:**
  - 하루 권장 급여량 (g)
  - 하루 총 kcal
  - 1회 급여량 (g) × 급여 횟수 (회/일)
  - 주의 문구 (예: "퍼피는 소량씩 자주 급여하는 것이 좋아요.")
  - 면책 고지: "수의사와 상담 후 급여량을 조정하세요."
- **예외 케이스:**
  - 칼로리 정보 없는 상품: "칼로리 정보가 없어 계산이 불가합니다." + 칼로리 직접 입력 옵션
  - 체중 0 입력: 입력 오류 안내
  - 극단적 체중값 (예: 강아지 100kg): 경고 및 재입력 유도

- **MVP 포함 여부:** ✅ 포함

### 8-1. 바코드 스캔

  3. 바코드 인식 시 자동으로 상품 DB를 조회한다.
  4. 상품이 있으면 분석 리포트 화면으로 이동한다.
- **입력값:** 카메라 바코드 (EAN-13, QR 등)
- **출력값:** 매칭된 상품 정보 + 분석 리포트
- **예외 케이스:**
  - 바코드 인식 실패: 재스캔 안내 또는 성분표 OCR 전환 유도
  - 미등록 상품 바코드: "아직 등록되지 않은 상품이에요. 성분표를 직접 촬영해보세요." 안내 + 상품 등록 요청 폼
- **우선순위:** P1

### 8-2. 성분표 OCR 촬영

- **목적:** 카메라로 사료 포장의 성분표를 촬영하면 OCR로 성분을 추출하여 분석한다.
- **사용자 시나리오:**
  1. 스캐너 탭 > "성분표 촬영" 선택한다.
  2. 성분표 영역을 카메라로 비추고 촬영한다.
  3. OCR 처리 후 추출된 성분 텍스트가 표시된다.
  4. 사용자가 오류를 수정 후 "분석하기" 버튼을 탭한다.
  5. 분석 리포트가 생성된다.
- **입력값:**
  - 카메라 이미지 또는 갤러리 이미지
  - OCR 결과 수정 텍스트 (선택)
- **출력값:**
  - OCR 추출 텍스트 (수정 가능 텍스트필드)
  - 성분 분석 리포트
  - 분석 이력 저장 (자동)
- **예외 케이스:**
  - OCR 인식률 낮음 (흐림, 반사, 소문자 밀집): "인식이 어려워요. 밝은 곳에서 다시 촬영해보세요." + 결과 수정 폼 제공
  - 성분표 아닌 영역 촬영: "성분표가 인식되지 않습니다. 성분표 부분을 정확히 비춰주세요."
  - 갤러리 이미지 권한 미허용: 권한 요청 다이얼로그
- **우선순위:** P1
- **MVP 포함 여부:** ✅ 포함

- **출력값:** 분석 이력 목록 (날짜, 상품명 또는 OCR 이미지 섬네일, 등급)

## 10. 비교함 기능

### 10-1. 상품 비교

- **목적:** 최대 4개 상품을 나란히 비교하여 최적 상품 선택을 돕는다.

- **출력값:**
  비교 항목별 테이블:

  | 항목 | 설명 |
  |------|------|
  | 가격 | 판매가 |
  | g당 가격 | 가성비 지표 |
  | 평점 | 별점 수치 |
  | 리뷰 수 | 신뢰도 지표 |

  | 성분 안전도 | 등급 및 점수 |
  | 최종 등급 | A++~C |
  | 주의 성분 | 목록 나열 |
  | 알러지 충돌 | 충돌 성분 표시 |
  | 주요 성분 | 상위 5가지 |
  | 권장 급여량 | 마이펫 기준 g/일 |

  - 항목별 최우수 상품 셀 강조 (초록 배경)
  - "이 상품 구매하기" 버튼
  - 비교함 저장 (URL 또는 링크 공유)

- **예외 케이스:**
  - 상품 1개만 담겨 있을 때 비교 시도: "2개 이상 상품을 추가해야 비교할 수 있어요."
  - 비교 항목 데이터 누락 상품: 해당 셀 "정보 없음" 표시
  - 비교 공유 링크 클릭 후 비로그인: 비교 화면 진입 후 가입 유도

- **예외 케이스:**
  - 검색어 미일치: 유사 성분명 제안
  - 미등록 성분 검색: "아직 등록되지 않은 성분입니다. 요청해주시면 검토 후 추가할게요."
- **우선순위:** P1
- **MVP 포함 여부:** ✅ 포함 (기본 성분 DB 300개 이상 우선 구축)

## 14. 알림 기능

---

### 14-1. 푸시 알림

- **목적:** 사용자가 관심 있는 상품과 활동에 대한 알림을 받아 앱 재방문율을 높인다.
- **사용자 시나리오:**
  1. 설정 > 알림 설정에서 항목별 on/off 설정한다.
  2. 해당 이벤트 발생 시 푸시 알림을 수신한다.
  3. 알림 탭에서 수신된 알림 이력을 확인한다.
- **입력값:** 알림 항목별 on/off 설정
- **출력값:**

  | 알림 유형 | 트리거 조건 | 내용 |
  |-----------|------------|------|
  | 찜 상품 가격 변동 | 가격 X% 이상 변동 시 | "OO 상품 가격이 N% 내렸어요!" |
  | 급여 리마인더 | 설정 시간 | "OO의 밥 시간이에요!" |
  | 커뮤니티 댓글 | 내 게시글에 댓글 등록 | "새 댓글이 달렸어요." |
  | 분석 완료 | OCR 분석 완료 | "OO 성분 분석이 완료됐어요." |
  | 주의 성분 발견 | 찜 상품 성분 DB 업데이트 시 | "OO에서 주의 성분이 발견됐어요." |
  | 추천 상품 업데이트 | 주간 추천 갱신 | "이번 주 OO에게 딱 맞는 신상품이 있어요!" |

- **예외 케이스:**
  - 알림 권한 미허용: 앱 내 알림 배지로 대체, 권한 요청 배너 표시
  - 알림 과다 발송 방지: 동일 유형 알림 1일 1회 제한
  - 앱 포그라운드 상태: 인앱 토스트로 대체
- **우선순위:** P1
- **MVP 포함 여부:** 가격 변동 / 분석 완료 알림만 포함

## 15. 관리자 기능

---

### 15-1. 상품 관리

- **목적:** 관리자가 상품 DB를 등록/수정/삭제하여 서비스 데이터 품질을 유지한다.
- **사용자 시나리오:**
  1. 관리자 웹 콘솔에 접속한다.
  2. 상품 목록에서 상품을 검색하고 수정한다.
  3. 신규 상품을 등록하거나 단종 상품을 삭제한다.
- **입력값:**
  - 상품명, 브랜드, 카테고리, 가격, 중량, 이미지 URL
  - 전성분, 보증성분, 칼로리
  - 연령대, 종 구분
  - 구매 링크
- **출력값:** 상품 DB 반영, 앱 내 즉시 적용
- **예외 케이스:**
  - 중복 상품 등록 시도: 유사 상품 목록 표시 후 확인 요청
  - 필수 항목 누락: 항목별 오류 표시 및 저장 불가
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함

---

### 15-2. 성분 위험도 관리

- **목적:** 성분 안전도 DB를 관리하여 분석 신뢰도를 유지한다.
- **사용자 시나리오:**
  1. 성분 관리 메뉴에서 성분을 검색한다.
  2. 성분의 위험도 등급, 설명, 기준값을 수정한다.
  3. 저장 시 앱 내 분석 결과에 즉시 반영된다.
- **입력값:** 성분명, 카테고리, 위험도 등급, 설명, 권장/주의 기준값
- **출력값:** 성분 DB 업데이트, 해당 성분 포함 상품의 분석 점수 재계산 트리거
- **예외 케이스:**
  - 위험도 변경 시 영향 받는 상품 수 미리 표시 후 확인 요청
- **우선순위:** P0
- **MVP 포함 여부:** ✅ 포함

---

### 15-3. 리뷰 / 커뮤니티 신고 관리

- **목적:** 부적절한 리뷰 및 신고된 커뮤니티 게시글을 검토하고 처리한다.
- **사용자 시나리오:**
  1. 관리자 콘솔에서 신고 목록을 확인한다.
  2. 게시글을 검토하고 삭제/경고/무시를 선택한다.
  3. 처리 결과를 신고자에게 알림으로 발송한다.
- **입력값:** 신고 ID, 처리 액션 (삭제 / 경고 / 무시 / 사용자 제재)
- **출력값:** 처리 결과 반영, 신고자 알림, 처리 이력 로그 기록
- **예외 케이스:**
  - 대량 신고 폭격 (특정 사용자 집중 신고): 어뷰징 감지 필터 적용
- **우선순위:** P2
- **MVP 포함 여부:** ❌ MVP 이후 (커뮤니티 기능 런칭과 동시)

---

### 15-4. 수의사 인증 관리

- **목적:** 수의사 인증 신청을 검토하고 승인/거절한다.
- **사용자 시나리오:**
  1. 관리자가 인증 신청 목록을 확인한다.
  2. 면허증 이미지와 번호를 검토한다.
  3. 승인 또는 거절 처리한다.
- **입력값:** 신청 ID, 처리 결과 (승인/거절), 거절 사유
- **출력값:** 사용자 배지 부여 또는 거절 알림 발송
- **예외 케이스:**
  - 면허 진위 확인 불가 시: "추가 서류 요청" 상태로 보류
- **우선순위:** P2
- **MVP 포함 여부:** ❌ MVP 이후

---

### 15-5. 랭킹 가중치 / 배너 / 추천 카테고리 관리

- **목적:** 랭킹 산출 기준 조정, 홈 배너 운영, 추천 카테고리 큐레이션을 통해 서비스 운영 유연성을 확보한다.
- **사용자 시나리오:**
  - 랭킹 가중치: 콘솔에서 항목별 % 수치를 조정하고 적용한다.
  - 배너: 이미지, 링크, 노출 기간을 설정하여 홈 배너를 등록한다.
  - 추천 카테고리: 홈 화면 추천 섹션의 카테고리 순서와 상품 목록을 큐레이션한다.
- **입력값:**
  - 랭킹 가중치: 항목별 비율 (합계 100% 검증)
  - 배너: 이미지, 링크 URL, 노출 시작/종료 날짜, 노출 위치
  - 추천 카테고리: 카테고리명, 연결 상품 목록, 노출 순서
- **출력값:** 즉시 앱 반영
- **예외 케이스:**
  - 가중치 합계 100% 초과: 저장 불가 오류
  - 배너 이미지 규격 불일치: 업로드 경고 + 리사이즈 옵션
- **우선순위:** P1
- **MVP 포함 여부:** ✅ 포함 (배너, 기본 추천 카테고리 관리만)
