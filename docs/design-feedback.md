# 베로로 디자인 피드백 (최종본)

# 디자인 원칙 (공통)

아래 피드백은 신규 디자인 제안이 아니라 기존 디자인을 개선하기 위한 피드백입니다.

## 유지할 요소

* 기존 디자인 시스템(컬러, 폰트, 컴포넌트 스타일)은 유지
* 현재의 Toss 스타일 기반 톤앤매너 유지
* UI를 새로 만드는 것이 아니라 기존 UI를 발전시키는 방향

## 개선 목표

* 사용자가 3초 안에 핵심 정보를 이해할 수 있도록 정보 계층 구조 개선
* 구매 전환율과 정보 전달력 향상
* 베로로의 핵심 가치인 "반려동물 맞춤 추천"이 더 잘 드러나도록 개선
* 쇼핑몰이 아닌 반려동물 건강/추천 플랫폼으로 인식될 수 있도록 개선

## UX 원칙

* 중요 정보 → 보조 정보 → 상세 정보 순으로 정보 구조 정리
* 점수, 궁합, 추천 이유를 우선 노출
* 가격, 리뷰, 상세 성분은 후순위 배치
* 모바일 사용성을 고려한 여백, 버튼 크기, 터치 영역 최적화
* Apple Human Interface Guidelines 및 Material Design UX 원칙 참고
* 자연스럽고 직관적인 인터페이스 구성

# 공통

## 디자인 시스템 통일

### 수정사항

* 전체적으로 모든 카드 및 박스가 너무 각져 보임
* 현재보다 더 둥근 모서리(Radius) 적용
* 사이트 전체에 동일한 곡률 규칙 사용

### 이유

* 현재 토스 스타일을 참고하고 있지만 전체적으로 딱딱한 느낌이 있음
* 곡률이 통일되지 않으면 디자인 완성도가 떨어져 보일 수 있음

### 원하는 방향

* 모든 카드, 버튼, 입력창, 정보 박스에 동일한 Radius 시스템 적용
* 전체적으로 부드럽고 친근한 인상 강화

---

## 정보 계층 구조 개선

### 수정사항

* 사용자가 3초 안에 핵심 정보를 파악할 수 있도록 정보 우선순위 재정리

### 이유

* 현재는 정보량은 충분하지만 어떤 정보를 먼저 봐야 하는지 명확하지 않음
* 베로로의 핵심 가치는 쇼핑이 아니라 반려동물 맞춤 분석과 추천임

### 원하는 방향

* 점수 → 추천 이유 → 핵심 특징 → 상세 정보 순으로 구조 정리
* 사용자가 제품을 빠르게 이해하고 의사결정할 수 있도록 개선

# 홈

## 식단 건강 점수 표현 수정

### 수정사항

* "식단 건강 점수" 문구 변경 검토

### 이유

* 건강 상태를 진단하는 서비스처럼 오해될 수 있음
* 베로로의 핵심은 건강 진단이 아니라 제품 적합도 분석임

### 원하는 방향

* "현재 사료 궁합"
* "현재 식단 적합도"
* "우리 아이 맞춤 점수"

등의 표현 검토

## 점수 산정 근거 제공

### 수정사항

* 점수 영역 하단에 펼침/접힘 UI 추가

### 이유

* 사용자는 점수보다 왜 그런 점수가 나왔는지 궁금해함
* 점수 신뢰도를 높일 수 있음

### 원하는 방향

* 기본 상태는 접힘
* 클릭 시 점수 산정 이유 표시

예시

* 닭 알러지 회피 +30
* 관절 건강 적합 +25
* 체중 관리 적합 +20

---

## 카테고리 아이콘 개선

### 수정사항

* 사료 / 간식 / 배변 / 관절 등 카테고리 아이콘 교체 검토

### 이유

* 현재 아이콘이 카테고리 의미를 직관적으로 전달하지 못함

### 원하는 방향

* 카테고리와 더 직관적으로 연결되는 아이콘 사용
* 전체 스타일 일관성 유지

# 검색

## 가격 강조도 축소

### 수정사항

* 가격 강조 감소
* 궁합 점수 강조 증가

### 이유

* 현재 검색 결과가 쇼핑몰 또는 가격 비교 사이트처럼 보임
* 베로로는 판매 플랫폼이 아니라 맞춤 추천 플랫폼임

### 원하는 방향

* 가격 위치에 궁합 점수 배치
* 가격은 보조 정보로 처리
* 작은 크기 + 흐린 텍스트 사용

---

## 가격 표기 방식 개선

### 수정사항

* 단일 가격 대신 가격 범위 또는 최저가 기준 표기 검토

### 이유

* 판매처마다 가격이 다르기 때문에 현재 표기는 현실성이 떨어짐
* 현재 가격 데이터가 어떤 기준으로 표시되는지 사용자가 이해하기 어려움

### 원하는 방향

* 42,900원~
* 최저가 기준

등의 방식 검토

### 추가 검토 사항

* 실시간 가격 연동 여부
* 가격 데이터 수집 방식
* 최저가 기준 노출 여부
* 가격 업데이트 주기

# 스캐너

## 스캔 가이드 명확화

### 수정사항

* 가이드 영역 시인성 향상

### 이유

* 현재 화면이 다소 어둡고 촬영 영역이 명확하지 않음
* 사용자가 어디에 성분표를 맞춰야 하는지 직관적으로 이해하기 어려움

### 원하는 방향

* 카메라 화면 위에 가이드 UI 오버레이
* 경계선 강조
* 촬영 위치를 직관적으로 인지 가능하도록 개선
* 가이드 외부 영역에도 실시간 카메라 화면 유지
* 포켓몬고 AR처럼 카메라 영상 위에 UI가 올라가는 구조 검토

---

## 점수 체계 역할 분리

### 수정사항

* 현재 동일한 의미의 궁합 점수가 중복 노출됨

### 이유

* 사용자가 점수의 의미를 구분하기 어려움
* 두 점수의 역할과 기준이 명확하게 구분될 필요가 있음

### 원하는 방향

우측 하단 점수
→ 마이펫 기준 궁합 점수

카드 중앙 하단 점수
→ 제품 자체 평가 점수

로 역할과 위치를 명확하게 분리

예시

제품 자체 평가 점수: 92점
베로와의 궁합 점수: 96점

# 분석 리포트

## 점수 산출 근거 시각화

### 수정사항

* 점수 구성 요소 추가

### 이유

* 현재는 점수는 보이지만 산출 근거가 명확하지 않음
* 점수의 투명성과 신뢰성을 높일 필요가 있음

### 원하는 방향

Progress Bar 또는 카드 형태

예시

* 영양성분 40/40
* 안전성 30/30
* 원료 품질 20/20
* 기호성 10/10
